以下の手順でPull request(PR)のレビューコメントを確認し、コードに反映してください。

1. 現在のブランチのPR確認
2. PRの差分確認
3. PRコード外レビューコメントの内容確認
4. PRコード内レビューコメントの内容確認
5. 対象ファイルの編集

# 現在のブランチのPR確認

以下のコマンドを使用して、現在のブランチに紐づくPRの情報を確認します。

```bash
gh pr view
```

# PRの差分確認

以下のコマンドを使用して、対象PRの変更ファイルと変更内容を確認します。

```bash
gh pr diff
```

# PRコード外レビューコメントの内容確認

以下のコマンドでPRのコード外レビューコメントの内容を確認してください。

```bash
gh pr view --comments
```

# PRコード内レビューコメントの内容確認

以下のコマンドでPRのコード内レビューコメントの内容を確認してください。

```bash
gh api "repos/{owner}/{repo}/pulls/$(gh pr view --json number --jq '.number')/comments"
```

# 対象ファイルの編集

収集したレビューコメントを以下の観点で精査し、実装すべき変更を特定してファイルを編集してください。

- 修正が必要なバグや問題の指摘
- コードスタイルや設計に関する提案
- 機能の追加・削除・変更に関する指摘
- 賞賛や質問のみのコメントなど、実装に直接関係しないものは除外する

各レビューコメントが指摘するファイルを読み取り、レビュー内容に基づいた修正を行います。
